# Public claim policy

1. A headline claim requires a public source, an explicit scope, and a visible boundary.
2. Repository tests are not independent validation when authored or maintained by the same project team.
3. A passing workflow reports repository state; it does not certify safety, quality, adoption, or production readiness.
4. Missing or unavailable evidence is `unknown`, never success.
5. Founder, maintainer, builder, and contributor are distinct roles.
6. Simulated challenges in the Proof Loom are explanatory and never presented as real repository execution.
7. External reproductions require manual review before any public evidence status changes.
8. Disputed or stale claims should remain visible until corrected, narrowed, or removed.
