# Design rationale

## Goal

Make the profile memorable because it behaves like Kartik's work: useful, inspectable, current, and open to challenge. The profile should not rely on inflated titles, activity theatre, generic AI language, or unsupported impact claims.

## Positioning choice

The package avoids the word **founder** in the headline. Starting and operating RGIT Rozgar remains visible through the factual phrase **built and maintain**. This communicates ownership without asking the visitor to reward a self-assigned title.

Recommended identity:

> Computer Engineering student building useful software for communities and systems that need to be inspectable.

This is intentionally quieter than “AI systems founder” or “100x engineer.” The repositories should create the stronger conclusion.

## Why this is interesting

The profile has five distinctive behaviors:

1. A mission-control graphic generated from real repository metadata.
2. A public claim-challenge workflow.
3. A reproduction path a visitor can run.
4. A failure log that shows how uncertainty changed the work.
5. Boundaries beside every ambitious claim.

These are not decorative widgets. They make the profile itself a small evidence system.

## Human voice rules

- Prefer “I built,” “I maintain,” and “I am testing” to broad identity labels.
- Use concrete nouns and verbs.
- Include limitations that are slightly inconvenient.
- Avoid symmetrical marketing copy where every project sounds equally mature.
- Do not list projects that have no public artifact as shipped.
- Do not publish adoption, user, benchmark, or security numbers without a stable source.
- Keep one or two sentences that sound personally written, such as “I like ambitious systems. I trust measured ones.”

## Visual restraint

The banner uses a systems-console language without pretending to be a real operational control room. No generated robot, galaxy, matrix rain, trophy wall, visitor counter, contribution snake, animated typing, or oversized technology grid is included.

## What to verify before publishing

- Kartik's exact role in Kairo-Phantom and the fork relationship.
- Whether Project X-Ray's stated release exists as a GitHub Release.
- Whether RGIT Rozgar's production repository will be renamed.
- Every test count and benchmark link.
- Gumloop cohort wording and current participation status.
- Links for community projects that are public.
