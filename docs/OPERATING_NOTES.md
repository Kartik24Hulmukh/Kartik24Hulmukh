# Operating notes

## Public claim policy
A claim belongs in the profile only when it has a source, a scope, and a visible boundary. CI success reports repository state; it does not certify product quality, safety, adoption, or independent validation.

## Role language
- **Founder** is reserved for RGIT Rozgar.
- **Lead Builder / Maintainer** describes hands-on Kairo-Phantom work while preserving the upstream relationship.
- **Contributor** is used where work is shared or exact ownership is narrower.

## Failure log
- Early RGIT Rozgar work prioritized presentation before governance. Permissions, review workflows, policies, security controls, and tests became foundations.
- Fixture-backed workflows can be mistaken for general live-GUI reliability. Live GUI capability remains experimental where not publicly demonstrated.
- Repository benchmarks are not independent validation. Third-party reproductions stay a separate evidence status.

## Dynamic content
Only allowlisted repository revision, CI, release, and timestamp fields are automated. Identity, roles, maturity, limitations, and impact remain manual.
